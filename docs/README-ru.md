# Rico

**Remote Claude Code Operator**

[English](README-en.md) | [한국어](../README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md) | [Español](README-es.md) | [Português (BR)](README-pt-br.md) | [Français](README-fr.md) | Русский | [Deutsch](README-de.md)

### Q. Что это?

**A. Я сделал это, потому что хотел работать с Claude Code лёжа.**

Это PWA-клиент, который позволяет использовать Claude Code на мобильных устройствах. Он связывается с Claude Code CLI через мост-сервер на Go и обеспечивает быстрые ответы через WebSocket-соединение в реальном времени.

![Rico Screenshots](../screenshots/rico-mockup.png)

## Возможности

- **Mobile-First PWA**: Установите на главный экран и используйте как нативное приложение
- **Чат в реальном времени**: Общение в реальном времени на основе WebSocket
- **Push-уведомления**: Push-оповещения при поступлении новых сообщений
- **Файловый менеджер**: Просмотр файловой системы сервера и файлов
- **Управление сессиями**: Сохранение и управление сессиями разговоров
- **Система SOUL**: Настраиваемая персона ИИ
- **Skills**: Расширенные функции через слэш-команды
- **i18n**: Многоязычная поддержка (корейский/английский)

### Почему нужны HTTPS и домен

Основные функции PWA (Service Worker, push-уведомления, установка на главный экран) **работают только через HTTPS** по соображениям безопасности. Кроме того, для получения SSL-сертификатов необходим **домен**.

Существует несколько способов получить домен и SSL-сертификаты, но этот проект использует Tailscale. Вы также можете использовать другие методы (Cloudflare Tunnel, ngrok, собственный домен и т.д.).

## Настройка Tailscale (метод, использованный в этом проекте)

Tailscale — это VPN-сервис, который также бесплатно предоставляет домены и SSL-сертификаты. Это удобно, когда нужна простая настройка HTTPS для личных проектов.

### 1. Установка Tailscale

**ПК (компьютер, на котором будет работать сервер):**
- Установите с https://tailscale.com/download для вашей ОС
- Войдите после установки (Google, GitHub и т.д.)

**Мобильное устройство:**
- Установите "Tailscale" из App Store / Play Store
- Войдите с **той же учётной записью**

### 2. Проверка домена

После установки Tailscale выполните в терминале:

```bash
tailscale status
```

Пример вывода:
```
100.94.195.110  your-machine    your-email@...
```

Формат домена: `your-machine.tail1234.ts.net`

> Также можно проверить в Консоли администратора Tailscale (https://login.tailscale.com/admin)

### 3. Получение SSL-сертификата

```bash
# Получить сертификат (бесплатно)
tailscale cert your-machine.tail1234.ts.net
```

Сгенерированные файлы:
- `your-machine.tail1234.ts.net.crt` (сертификат)
- `your-machine.tail1234.ts.net.key` (закрытый ключ)

Скопируйте эти файлы в папку `server/certs/`.

## Установка

### 1. Клонирование

```bash
git clone https://github.com/Epsilondelta-ai/rico.git
cd rico
```

### 2. Генерация VAPID-ключей

Сначала сгенерируйте VAPID-ключи для push-уведомлений:

```bash
npx web-push generate-vapid-keys
```

Пример вывода:
```
=======================================

Public Key:
BNlx...your_public_key...

Private Key:
abc1...your_private_key...

=======================================
```

Сохраните эти ключи. Они будут использоваться в настройке ниже.

### 3. Настройка сервера

```bash
cd server
go mod download
cp .env.example .env
```

Отредактируйте `server/.env`:
```env
# VAPID-ключи (сгенерированные выше)
VAPID_PUBLIC_KEY=BNlx...your_public_key...
VAPID_PRIVATE_KEY=abc1...your_private_key...

# Порт сервера
SERVER_PORT=8080

# SSL-сертификат (пути к файлам от Tailscale)
SSL_CERT_FILE=./certs/your-machine.tail1234.ts.net.crt
SSL_KEY_FILE=./certs/your-machine.tail1234.ts.net.key
```

### 4. Настройка фронтенда

```bash
cd ../web
npm install
cp .env.example .env
```

Отредактируйте `web/.env`:
```env
# Адрес API-сервера (используйте домен Tailscale)
VITE_API_BASE=https://your-machine.tail1234.ts.net:8080

# Публичный VAPID-ключ (тот же, что в server/.env)
VITE_VAPID_PUBLIC_KEY=BNlx...your_public_key...
```

### 5. Сборка и запуск

Выполните из корня проекта (`rico/`).

**Windows:**
```bash
scripts\run-windows.bat
```

**Linux/macOS:**
```bash
chmod +x scripts/run-linux.sh   # только в первый раз
./scripts/run-linux.sh
```

Скрипт выполняет по порядку: установка зависимостей → сборка фронтенда → сборка сервера → запуск сервера.

Лог при успешном запуске:
```
Rico 설정 로드 완료:
  - RICO_BASE_PATH: /path/to/rico
  - SERVER_PORT: 8080
Rico 브릿지 서버 시작 (HTTPS): :8080
```

### Доступ с мобильного

1. Запустите приложение Tailscale на мобильном (проверьте подключение)
2. Откройте `https://your-machine.tail1234.ts.net:8080` в браузере

**iOS (Safari):**
3. Нажмите кнопку «Поделиться» → Выберите **«На экран «Домой»»**
4. Установка PWA завершена

**Android (Chrome):**
3. Меню (⋮) → Выберите **«Добавить на главный экран»** или **«Установить приложение»**
4. Установка PWA завершена

> Этот проект был протестирован в iOS Safari. Ожидается, что он будет работать на Android, но не тестировался.

## Кастомизация

### SOUL (Персона ИИ)

Вы можете настроить личность, стиль речи и поведение ИИ, отредактировав `context/personas/default/SOUL.md`.

```markdown
# SOUL

You are a Claude Code agent. Respond naturally.
```

Настройка по умолчанию содержит минимум инструкций, позволяя Claude отвечать естественно. Вы можете добавить детальные персоны по мере необходимости.

### Язык (i18n)

Вы можете изменить язык в меню настроек приложения. В настоящее время поддерживаются корейский и английский.

**Добавление нового языка:**

1. Создайте `{код_языка}.json` в папке `web/src/locales/` (например: `ja.json`)
2. Скопируйте структуру `ko.json` и переведите
3. Отредактируйте `web/src/lib/i18n.ts`:
   ```typescript
   import ja from '../locales/ja.json';
   addMessages('ja', ja);
   ```
4. Обновите UI переключения языка (SessionListScreen.svelte)

## Устранение неполадок

### Push-уведомления не приходят

- Проверьте, правильно ли настроены VAPID-ключи
- Убедитесь, что **публичные ключи идентичны** в `server/.env` и `web/.env`
- Проверьте, разрешены ли уведомления в браузере

### HTTPS-соединение не работает

- Убедитесь, что Tailscale подключён и на ПК, и на мобильном
- Проверьте правильность путей к файлам SSL-сертификата
- Проверьте, не истёк ли сертификат (требуется обновление каждые 3 месяца)
  ```bash
  tailscale cert your-machine.tail1234.ts.net  # перевыпуск
  ```

### Ошибка «Не удаётся подключиться к сайту»

- Проверьте, запущен ли сервер
- Убедитесь, что приложение Tailscale подключено на мобильном
- Проверьте правильность адреса домена

### Нет ответа от Claude Code

- Убедитесь, что выполнен вход через `claude login`
- Проверьте логи сервера на наличие ошибок: `server/logs/`
- Проверьте, работает ли Claude Code CLI:
  ```bash
  claude "Hello"
  ```

## Разработка

### Локальная разработка (HTTP)

Для локального тестирования без SSL-сертификатов:

**Терминал 1 — Сервер:**
```bash
cd server
# Закомментируйте SSL_CERT_FILE, SSL_KEY_FILE в .env
go run main.go
```

**Терминал 2 — Фронтенд:**
```bash
cd web
npm run dev
```

> Примечание: В режиме HTTP push-уведомления, установка PWA и т.д. работать не будут.
